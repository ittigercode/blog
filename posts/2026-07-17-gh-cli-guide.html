🖥️ Windows 安装 GitHub CLI 完全指南

2026-07-17 · 四种安装方式 + 横向对比
GitHub CLI（gh）是 GitHub 官方命令行工具。本文对比四种 Windows 安装方式，分析 winget 管理员/普通用户的区别，并回答一个根本问题：gh CLI 到底有什么用？

一、横向分析：四种安装方式对比

方式命令难度推荐指数

🔵 Scoopscoop install gh⭐⭐★★★★

方式一：winget（推荐）

Windows 自带，不需要装任何东西：

winget install --id GitHub.cli

装完关掉重开 PowerShell，gh --version 验证。

方式二：Scoop

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
irm get.scoop.sh | iex
scoop install gh

方式三：Chocolatey

choco install gh

方式四：手动解压

下载 zip（非 msi）：github.com/cli/cli/releases
选 gh_X.X.X_windows_amd64.zip，解压到 C:\gh\，加到系统 PATH。

二、winget 管理员 vs 普通用户

维度普通用户管理员
安装位置%LOCALAPPDATA%C:\Program Files\
影响范围只装给自己这台电脑所有人
UAC 弹窗不需要需要点「是」
PATH自动加（可能不全）自动加，全局生效
建议用普通用户跑就行。省事，不弹窗，效果完全一样。

三、gh CLI 有什么用？

gh 不只是「另一个 Git 工具」。它把 GitHub 的整个 Web 操作搬到了终端：

3.1 认证管理

gh auth login          # 浏览器授权，比 SSH/Token 都简单
gh auth status         # 查看登录状态

3.2 仓库操作

gh repo create my-project --public   # 创建仓库
gh repo clone owner/repo             # 克隆
gh repo view owner/repo --web        # 浏览器打开

3.3 Issue 和 PR

gh issue create --title "Bug: xxx" --body "..."
gh issue list
gh pr create --title "Feature" --body "..."
gh pr review --approve
gh pr merge

3.4 CI/CD

gh run watch           # 实时看 Actions 日志
gh run list            # 查看工作流历史

3.5 Release

gh release create v1.0 --title "正式版" --notes "更新内容"

四、和 Git 的关系

很多人混淆 gh 和 git：

gitgh
定位分布式版本控制GitHub 平台操作
操作范围任何 Git 仓库仅 GitHub
典型命令commit, push, mergeissue, pr, release
替代关系互不替代，是互补
git 管代码版本，gh 管 GitHub 平台——各司其职，配合使用。

五、为什么需要 gh？

三种 GitHub 认证方式的横向对比：

设置难度密钥生成→上传网页生成Token一行 login
安全性★★★★★★★★★★★★
WSL 稳定性★★★★★★★（TLS 常挂）★★★★
Issue/PR 操作不支持不支持支持

最佳实践： git push/pull 用 SSH（稳定），Issue/PR/Release 用 gh CLI（方便）。两者互补，不是替代。
