🔑 Hermes 模型 API Key 配置完全指南

2026-07-23 · 从千问到全模型，一篇讲透
本文以千问（Qwen/DashScope）API Key 为切入点，手把手教你申请密钥、配置到 Hermes、让 Skill 调用——最后横向扩展到 OpenAI、DeepSeek、Anthropic 等所有主要模型的配置异同。

一、申请千问（DashScope）API Key

1.1 三步拿到 Key

  1. 打开 dashscope.aliyun.com，用阿里云账号登录
  2. 右上角头像 → API-KEY 管理
  3. 点 创建 API-KEY → 立即复制保存（只显示一次！）

1.2 创建工作空间

千问画图（Qwen Image）还需要工作空间：

  1. 打开 dashscope.console.aliyun.com
  2. 创建或选择一个 工作空间
  3. 记下 Workspace ID（页面左上角）
💰 费用：新用户有免费额度。千问文本模型（qwen-turbo/plus/max）很便宜，画图稍贵但试手够用。

二、Hermes 配置千问 API Key

2.1 核心：token 填在哪里

Hermes 不强制要求配置 API Key——为「自带密钥」模型提供 API Key 即可使用。所有模型的 API Key 统一放在：

~/.hermes/.env

直接用你喜欢的文本编辑器打开这个文件，添加以下内容：

# 千问 / DashScope
DASHSCOPE_API_KEY=***   # 你的 API Key
DASHSCOPE_WORKSPACE_ID=你的工作空间ID
DASHSCOPE_REGION=cn-beijing            # 或 singapore
环境变量加载时机：每次 Hermes 启动时读取。修改后需要重启（/reset 或重新运行 hermes chat）。运行时也可用 /reload 重新加载。

为什么其他 Key 也要放 .env 而不是 config.yaml？ config.yaml 是配置文件，可以提交到版本控制。.env 是敏感信息，Hermes 会阻止 AI 读取它（防泄露），且通常被 .gitignore。

三、Skill 如何使用千问 API Key

3.1 Skill 不会主动读 .env

Skill 本身只是一个 Markdown 文档，它不直接读环境变量。真正调用 API 的是 Skill 里的 Python 脚本：

# Skill 里的脚本按需读取环境变量
python3 scripts/generate_image.py --preflight
# 脚本内部检查 os.environ.get("DASHSCOPE_API_KEY")
# 退出码 5 = 缺少 Key，退出码 0 = 已配置

3.2 以手绘幻灯片 Skill 为例

手绘幻灯片包（handdrawn-slide-skill-pack）的 Qwen 路径执行流程：

# 1. 预检：只检查依赖和Key，不联网不收费
python3 scripts/generate_image.py --preflight

# 2. 试跑：验证请求格式，不联网
python3 scripts/generate_image.py --request page.json --output page.png --dry-run

# 3. 正式生成：发起计费API调用
python3 scripts/generate_image.py --request page.json --output page.png

关键设计： Skill 脚本从当前会话的环境变量中读取 Key，不把 Key 写在 Skill 目录里。这样 Skill 可以分享给任何人，不会泄露秘钥。

4.3 千问的特殊性

千问是少数需要三个环境变量的模型：

DASHSCOPE_API_KEY — 认证密钥（所有模型都需要）
DASHSCOPE_WORKSPACE_ID — 千问特色，类似 OpenAI 的 Organization ID
DASHSCOPE_REGION — 数据节点选择（beijing / singapore）

大部分模型只需要一个 KEY 变量。千问多出来的两个变量是因为阿里云的工作空间架构和地域节点设计。

五、Hermes 高级配置

5.1 切换默认模型

hermes model          # 交互式选择
hermes chat -m deepseek/deepseek-chat   # 临时指定

5.2 配置多 Provider

在 ~/.hermes/config.yaml 中：

model:
  default: "deepseek/deepseek-chat"
  provider: "deepseek"

5.3 使用 OpenRouter（一个 Key 调用所有模型）

# .env
OPENROUTER_API_KEY=***   # 一个 Key 就能用 200+ 模型

# 使用
hermes chat -m anthropic/claude-sonnet-4
hermes chat -m google/gemini-2.5-pro
推荐：如果你不想每个模型都申请 Key，用 OpenRouter。一个 Key，调用 200+ 模型，按量付费，不用在各个平台之间切来切去。

六、总结

层次要点
🔑 申请各模型官网 → API Key 管理 → 创建 → 复制
⚙️ 配置填到 ~/.hermes/.env，变量名参照上表
🎨 Skill脚本通过 os.environ 读取，.env 统一管理
🔄 通用性所有模型遵循同一套 .env 范式，差异仅在于变量名和数量
核心原则：绝不在代码或 Skill 目录里硬编码 Key。所有秘钥统一由 ~/.hermes/.env 管理，Hermes 自动加载、自动防泄漏。换模型只需改一行环境变量。
