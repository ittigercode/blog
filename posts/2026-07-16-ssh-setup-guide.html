🔐 Git SSH 配置终极指南

2026-07-16 · 3 分钟搞定 · 永久生效
告别密码输入和 TLS 报错。用 Ed25519 + SSH，一次配置，终身受益。

为什么要用 SSH？

HTTPS 方式有三大痛点：

  • ⚠️ TLS 握手经常失败（尤其 WSL 环境）
  • ⚠️ Token 会过期，忘了更新就 push 不了
  • ⚠️ 每次操作都要验证，烦

SSH 的优势：密钥永不过期、不需要密码、TLS 不稳定的环境也能稳定推送。

第一步：生成 Ed25519 密钥

ssh-keygen -t ed25519 -C "your@email.com"

一路回车即可。密钥保存在 ~/.ssh/id_ed25519（私钥）和 ~/.ssh/id_ed25519.pub（公钥）。

⚠️ 为什么是 Ed25519？ 比 RSA 更短（256bit vs 4096bit）、更快、更安全。GitHub/GitLab/Bitbucket 三家都推荐。

第二步：添加公钥到 GitHub

cat ~/.ssh/id_ed25519.pub

复制输出内容 → 打开 github.com/settings/keys → New SSH Key → 粘贴 → 保存。

第三步：测试连接

ssh -T git@github.com

看到 Hi yourname! You've successfully authenticated 就成功了。

第四步：切换仓库到 SSH

cd 你的仓库
git remote set-url origin git@github.com:用户名/仓库名.git
git push

进阶：多密钥管理

如果你有 GitHub 和 GitLab 两个账号，用不同密钥：

# 生成两个密钥
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_github -C "github@email"
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519_gitlab -C "gitlab@email"

# 配置 ~/.ssh/config
Host github.com
  HostName github.com
  IdentityFile ~/.ssh/id_ed25519_github

Host gitlab.com
  HostName gitlab.com
  IdentityFile ~/.ssh/id_ed25519_gitlab

常见问题

Q: Permission denied (publickey)
→ 公钥没上传到 GitHub，或者 ssh-agent 没加载私钥。执行 ssh-add ~/.ssh/id_ed25519

Q: 端口 22 被防火墙封了
→ 用 HTTPS 端口：ssh -T -p 443 git@ssh.github.com

Q: 换了电脑怎么办
→ 在新电脑上重新生成密钥并上传。不要复制私钥，每台机器独立密钥更安全。

本文由实操经验总结，适用于 GitHub/GitLab/Gitee 等所有 Git 托管平台。
